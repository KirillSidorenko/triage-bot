# План веб-страницы

Цель страницы: быстро продать заказчику идею автоматизации triage обращений, не перегружая его внутренней механикой.

Главный принцип: меньше текста, больше визуального доказательства.

## 1. Hero

### Задача блока

За 5 секунд объяснить, какую бизнес-проблему решает продукт и какой даёт выигрыш.

### Текст

`AI Triage Bot`

`Убирает ручной triage входящих обращений и возвращает команде часы операционного времени.`

`Вместо 3-8 минут на первичный разбор тикета система обрабатывает обращение примерно за 3 секунды и передаёт людям только критичные и неоднозначные кейсы.`

### Визуал

- Предпочтительно autoplay hero-video с полным циклом обработки.
- Использовать: `demo/web/EasyCase-hero-1280.mp4`
- Poster: `demo/web/EasyCase-hero-poster.jpg`
- Fallback: `demo/screenshots/demo.gif`

### Дополнительно

- Подзаголовок должен быть в 2 строки максимум.
- Рядом или под GIF можно поставить 3 короткие метрики-карточки:
  - `~3 сек на triage`
  - `92% точность маршрутизации`
  - `94% review trigger accuracy`

## 4. Блок "Как это работает"

### Задача блока

Показать логику решения на одном экране без ухода в детали n8n-узлов.

### Заголовок

`Как работает решение`

### Текст

`Сообщение клиента поступает во входящий канал, AI определяет категорию и приоритет, создаёт задачу в трекере и отправляет рискованные кейсы на ручную проверку.`

Короткая схема в 4 шага:

1. `Входящее сообщение`
2. `AI triage`
3. `Задача в рабочем контуре`
4. `Review для рискованных кейсов`

### Визуал

- Основной скриншот процесса через реальный интерфейс.
- Использовать: `demo/screenshots/telegram-billing.png`
- Вторым экраном или справа:
  - `demo/screenshots/telegram-human-review.png`

### Дополнительно

- Здесь не писать про JSON, threshold и internal fields.
- Логика должна читаться с первого взгляда.

## Сценарий записи экрана для GIF или видео

### Цель демонстрации

Показать не “чат с ботом”, а полный операционный цикл:

1. клиент отправляет сообщение;
2. workflow в `n8n` принимает решение;
3. задача появляется в рабочем контуре;
4. результат фиксируется в логе;
5. критичные кейсы не уходят в full auto.

### Что важно учесть по текущему коду

- Подтверждение клиенту в Telegram сейчас короткое и без issue key.
- Issue key и ссылка на тикет появляются в reviewer-уведомлении и в Google Sheets.
- `Yandex Tracker` должен быть в кадре как конечный operational destination в обеих ветках.
- Для review-сценария важно показать не только review queue, но и escalation/reviewer message в Telegram.
- В коде есть две ключевые ветки, которые стоит показать:
  - `Auto Route`
  - `Needs Review`

### Что подготовить перед записью

- Открытый чат с ботом в Telegram.
- Открытый workflow/execution view в `n8n`.
- Открытый `Yandex Tracker` с нужной очередью.
- Открытый `Google Sheets` лог.
- Открытый reviewer-чат в Telegram для сценария review.

### Сценарий для короткой GIF

Лучший вариант для GIF: показать только happy path без лишних переключений.

Сообщение:

`Мне дважды списали деньги за подписку в этом месяце. Верните деньги!`

Ожидаемый путь по коду:

- категория: `billing`
- приоритет: `high`
- ветка: `Auto Route`
- `needs_review = false`

Последовательность кадров:

1. В Telegram отправляется сообщение клиента.
2. Сразу после этого короткий переход на `n8n` execution.
3. Видно, что workflow проходит через `DeepSeek - Classify` -> `Needs Review?` -> `Yandex Tracker - Auto Route`.
4. Переход в `Yandex Tracker`, где появился новый тикет в billing queue.
5. Открывается сама карточка тикета, чтобы было видно:
   - summary;
   - queue;
   - описание с исходным сообщением клиента.
6. Переход в `Google Sheets`, где появилась строка с:
   - `category = billing`
   - `priority = high`
   - `needs_review = false`

Что должно остаться на экране в конце GIF:

- новый тикет в очереди;
- открытая карточка тикета в `Yandex Tracker`;
- строка в логе;
- ощущение, что triage завершился без участия человека.

### Сценарий для полного видео

Для видео лучше показать 2 кейса: auto route и human review.

#### Сцена 1. Открытие

Что на экране:

- короткий титул;
- открытые окна: Telegram, `n8n`, `Yandex Tracker`, `Google Sheets`.

Текст за кадром:

`Покажу, как AI triage автоматически разбирает входящее обращение, создаёт задачу и передаёт рискованные кейсы на ручную проверку.`

#### Сцена 2. Автоматический маршрут

Сообщение:

`Мне дважды списали деньги за подписку в этом месяце. Верните деньги!`

Что показать:

1. Клиент отправляет сообщение в Telegram.
2. Бот отвечает стандартным подтверждением.
3. В `n8n` видно успешное прохождение по ветке `Auto Route`.
4. В `Yandex Tracker` появляется тикет в billing queue.
5. Открывается карточка тикета в `Yandex Tracker`, чтобы было видно summary и описание обращения.
6. В `Google Sheets` появляется строка с `billing`, `high`, `needs_review = false`.

Текст за кадром:

`Для понятного кейса workflow сам определяет категорию, приоритет и сразу отправляет задачу в рабочую очередь.`

#### Сцена 3. Критичный кейс и эскалация

Сообщение:

`Кто-то взломал мой аккаунт! Вижу входы из другой страны!`

Ожидаемый путь по коду:

- категория: `account_access`
- приоритет: `critical`
- ветка: `Needs Review`
- `needs_review = true`

Что показать:

1. Клиент отправляет сообщение в Telegram.
2. Бот отвечает подтверждением.
3. В `n8n` видно, что workflow ушёл в `Yandex Tracker - Needs Review`.
4. В `Yandex Tracker` тикет появляется в review queue, а не в финальной команде.
5. Открывается карточка тикета в `Yandex Tracker`, где видно:
   - review queue;
   - summary;
   - предварительную категорию;
   - предварительный приоритет;
   - причину ручной проверки.
6. В reviewer-чате Telegram приходит escalation message с:
   - оригинальным сообщением;
   - номером тикета;
   - предварительной категорией;
   - предварительным приоритетом;
   - кратким summary;
   - причиной проверки;
   - ссылкой на тикет.
7. В `Google Sheets` строка появляется с `needs_review = true`.

Текст за кадром:

`Критичные обращения не уходят в автоматическую маршрутизацию. Workflow создаёт тикет в review queue, а затем отправляет reviewer/escalation message с контекстом и ссылкой на карточку.`

#### Сцена 4. Закрывающий кадр

Что показать:

- `n8n` workflow на экране;
- рядом Tracker и лог;
- короткий финальный тезис.

Финальная фраза:

`n8n здесь не просто связывает сервисы. Он управляет всей цепочкой: приём сообщения, AI triage, создание тикета в Yandex Tracker, escalation message и аудит результата.`

### Рекомендация по формату

- Для hero GIF на сайте: `8-12 секунд`, только auto route сценарий.
- Для полной демо-записи: `35-60 секунд`, auto route + human review.
- Для GIF лучше не показывать таблицы целиком и не листать длинные интерфейсы.
- Для видео лучше делать крупные зумы на:
  - ветку `Auto Route`;
  - ветку `Needs Review`;
  - новую карточку в `Yandex Tracker`;
  - review queue в `Yandex Tracker`;
  - новую строку в Google Sheets;
  - reviewer/escalation message в Telegram.
