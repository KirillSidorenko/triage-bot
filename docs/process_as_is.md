# Customer Support: текущий процесс (As-Is)

## Описание процесса

Типичный ручной процесс обработки обращений клиентов в SaaS-компании среднего размера (50-200 сотрудников, 500-5000 обращений/месяц).

## Шаги процесса

### 1. Получение обращения
- Клиент пишет в support-канал (email, чат, форма на сайте, Telegram)
- Сообщение попадает в общую очередь / shared inbox
- Нет автоматической сортировки — всё в одном потоке

### 2. Первичный просмотр (L1 агент)
- Support-агент вручную читает сообщение
- Определяет категорию (billing? bug? account issue?)
- Определяет приоритет на основе опыта и интуиции
- **Среднее время:** 2-5 минут на одно обращение

### 3. Маршрутизация
- Агент решает, кому передать: billing team, engineering, account management
- Пересылает сообщение в нужный канал / ставит тег в тикет-системе
- При ошибке маршрутизации — обращение "гуляет" между командами
- **Среднее время:** 1-3 минуты

### 4. Обработка (L2 специалист)
- Специалист получает обращение, часто без контекста
- Перечитывает сообщение заново
- Решает проблему или эскалирует дальше

### 5. Ответ клиенту
- Специалист отвечает клиенту
- Закрывает тикет

## Боли текущего процесса

| Боль | Описание | Влияние |
|------|----------|---------|
| **Время на триаж** | 3-8 минут на каждое обращение только на классификацию и роутинг | При 100 обращений/день = 5-13 часов/день чистого триажа |
| **Inconsistency** | Разные агенты классифицируют по-разному | Один и тот же кейс может быть `billing` у одного агента и `complaint` у другого |
| **Ошибки маршрутизации** | 15-25% обращений уходят не в ту команду | Увеличивает время решения на 30-60 минут |
| **Потеря контекста** | При пересылке между командами теряется summary | Специалист тратит время на повторное чтение |
| **Нет приоритизации** | Critical issues обрабатываются в порядке очереди | Клиенты с urgent проблемами ждут наравне со всеми |
| **Нет аналитики** | Нет данных о распределении категорий, времени обработки | Невозможно оптимизировать процесс |

## Baseline-метрики

| Метрика | Текущее значение | Источник |
|---------|------------------|----------|
| Среднее время триажа | 5 минут | Industry benchmark |
| Routing accuracy | 75-85% | Industry benchmark |
| First response time | 2-4 часа | Industry benchmark |
| Manual touch rate | 100% | Все обращения обрабатываются вручную |
| Consistency rate | ~70% | Разные агенты → разная классификация |

## Объём

- **Малый SaaS:** 50-200 обращений/месяц
- **Средний SaaS:** 500-2000 обращений/месяц
- **Крупный SaaS:** 5000+ обращений/месяц

Даже для малого SaaS: 200 обращений × 5 мин = ~17 часов/месяц чистого триажа.
